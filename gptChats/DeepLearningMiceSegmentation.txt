## 1) Clarify the core learning target: semantic vs instance segmentation
Because you have **two mice**, the most important design decision is what the ground-truth masks encode:

### A. Ground truth distinguishes mouse identities (best-case)
If the partner video provides **two separate masks** (mouse A mask, mouse B mask) or a **single mask with labels {background, mouseA, mouseB}**, then you can train a **3-class semantic segmentation** network (or 2-channel instance mask output).  
This is the simplest path to stable identity over time.

**Output options**
- **3-class softmax**: background / mouseA / mouseB  
- **2-channel sigmoid**: one channel per mouse (+ implicit background)

Either works. Softmax is conceptually cleaner if pixels can only belong to one class.

### B. Ground truth is “union of both mice”
If the labels only indicate “mouse vs background” (no identity), then you have two choices:
- Train **mouse vs background** segmentation and do **instance separation** by connected components + tracking (works if mice rarely touch/overlap).
- Move to **instance segmentation** (heavier): e.g., embedding-based instance seg, Mask R-CNN style, or “two-mask” output with permutation-invariant loss. For only **two** animals, you can still do a two-head model with a **permutation-invariant matching loss** fairly cleanly, but it’s more fiddly than case A.

Given your description (“segments covered by the two mice”), I suspect you effectively have case A (two labeled segments). If so, exploit it.

## 2) Baseline that’s very likely to work: 2D U-Net per frame
Your instinct is right: start with a **per-frame 2D U-Net** (or U-Net with a modern encoder).

### Why this baseline is strong here
- Simulation means **clean labels** → segmentation converges quickly.
- With a laptop 4090, you can still train decent 2D segmentation at high throughput using mixed precision.
- Debugging is straightforward (you can visually inspect predictions frame-by-frame).

### Concrete architecture recommendations
- **U-Net with encoder**: ResNet34 / EfficientNet-B0/B1 / ConvNeXt-Tiny as encoder + U-Net decoder.
- Input: RGB frame (optionally + extra channels; see temporal hacks below)
- Output: 3 classes (bg, mouseA, mouseB) with softmax

### Loss + metrics
For two-object segmentation, I’d use a **compound loss**:
- **Dice loss** (per class, excluding background or with low weight)  
- + **Cross-entropy** (stabilizes early training)  
Optionally:
- **Focal loss** if class imbalance is annoying (usually background dominates).

Metrics:
- Per-class IoU/Dice
- Boundary F1 (optional if edges matter)
- “Identity swap rate” (if labels represent identity)

### Data splitting (important)
Split **by video**, not by frame. Otherwise you’ll leak near-identical frames into validation and get an overly optimistic score.

### Augmentation
Even simulated data benefits from augmentation; it prevents brittle models and helps generalize if you later change rendering conditions:
- Affine transforms (scale/rotate/translate)
- Motion blur (small)
- Brightness/contrast/gamma
- Noise / JPEG artifacts
- Random occlusions (small cutout) if you want robustness

## 3) Tracking: do it, but as a post-process first
You’re right that temporal continuity is available “for free”, but the highest ROI is usually:

### **Segmentation per frame → then tracking by association**
Given only **two mice**, you can do very robust tracking without deep temporal nets.

Pipeline:
1. Run segmentation on each frame (3-class output or two masks).
2. Extract per-mouse object(s) (connected component(s), largest blob per mouse class).
3. Compute features: centroid, area, maybe a small appearance embedding from the RGB crop.
4. Match objects frame-to-frame with:
   - **IoU matching** between masks, or
   - centroid distance + IoU cost
   - solve via Hungarian algorithm (overkill for 2 objects, but trivial to implement)
5. Optional smoothing with a **Kalman filter** on centroid/velocity.
6. Enforce constraints: “exactly two tracks”, “no impossible teleportation”.

This buys you:
- Stable identities even when the segmentation occasionally jitters.
- A clean interface to behavior analysis downstream.

### What about mice touching / crossing?
That’s the hard case. Still, for two animals you can handle most of it with:
- Using the **identity-labeled segmentation** (case A) so the network learns identity cues.
- A tracker that can survive short ambiguous intervals (keep last velocity + reassign when separated).
- A fallback rule: if the model outputs merged blobs or uncertain separation, rely on motion continuity rather than instantaneous segmentation.

## 4) “Temporal information” without paying the full 3D/Transformer cost
If you want to incorporate motion but keep compute close to 2D U-Net, there are two very practical middle paths.

### Option 1: Feed previous mask(s) as extra input channels
You train a 2D net that sees:
- current RGB frame
- previous frame’s mask(s) (ground truth during training, predicted during inference)

Input channels become: RGB (3) + prev_mask_onehot (e.g., 2 or 3 channels).

This often improves:
- temporal stability
- identity persistence near contact

Training trick:
- Start with “teacher forcing” (use GT previous mask)
- Then mix in predicted previous mask sometimes (scheduled sampling) to reduce train–test mismatch.

Compute cost is almost unchanged from the baseline.

### Option 2: Optical-flow warping of the previous mask
You can compute cheap optical flow (classical or a lightweight model) and warp the previous mask into the current frame, then provide it as input. This makes the “previous mask hint” align with motion and tends to help in fast movement.

## 5) Full temporal models (justified, but only after the baseline)
If you later decide the baseline+tracking isn’t good enough (typically because of prolonged close interaction/occlusion), then consider:

### A. Recurrent U-Net (ConvGRU/ConvLSTM)
Add a ConvGRU at the bottleneck or between decoder stages.
- Input: sequence of frames (e.g., 5–15)
- Output: mask at each timestep or last timestep

This captures motion continuity without 3D convolutions everywhere.

### B. 3D U-Net / 2.5D models
- 2.5D: stack N frames as channels (e.g., 5 frames → 15 channels RGB) and keep a 2D net.
- 3D: true spatiotemporal convs (heavier, more memory).

On a laptop GPU, 2.5D is usually the sweet spot if you insist on sequence input.

### C. Transformer-based video segmentation
Powerful but higher engineering and compute cost. I’d only go here if you end up needing SOTA temporal reasoning.

## 6) Laptop RTX 4090 training strategy (practical constraints)
Without assuming exact VRAM, laptop 4090s are commonly in the **~16GB** range. Either way, you can make training smooth:

- Use **mixed precision** (torch autocast + GradScaler).
- Use **reasonable resolution** first (e.g., 512×512 or 640×640). If videos are HD, downsample for training and optionally refine later.
- Use **patch training** if the cage is large but mice are small; sample crops around mice to increase effective batch size.
- If batch size is small, use:
  - gradient accumulation
  - GroupNorm instead of BatchNorm (or freeze BN in encoder)

## 7) How this connects to behavior recognition (running, rearing, interaction)
You didn’t explicitly ask for behavior classification yet, but segmentation+tracking is the foundation.

A clean next stage is:
- From tracked masks, compute per-frame features:
  - speed/acceleration
  - body orientation (via PCA on mask)
  - height proxy (rearing might change silhouette/area if the simulation encodes it visually)
  - inter-mouse distance, relative heading, overlap/contact metrics
- Then train:
  - a temporal classifier (TCN / BiLSTM / Transformer) on those features, or
  - a video model on cropped tubes around each mouse (more compute)

This modular approach is usually more controllable than trying to make one big end-to-end network do everything.

## 1) What exactly are we predicting under occlusion?
Two possibilities exist; your downstream expectations depend on which one your simulator provides:

- **Visible (modal) segmentation**: label marks only pixels actually visible for each mouse. If one mouse fully occludes the other in a view, the hidden mouse’s mask is (correctly) absent in those pixels.
- **Amodal segmentation**: label includes the occluded mouse’s full silhouette “behind” the occluder.

Most simulation pipelines output **modal** masks by default. Your sentence “predicting the partially occluded mouse segments” is consistent with modal segmentation (segment the *visible* part even if it’s small). That’s the easiest and most stable target. If it’s amodal, the problem becomes “shape completion under occlusion” and is still learnable, but you’ll want stronger priors/temporal cues and you should expect more ambiguity.

Everything below works for either, but the evaluation and expected behavior differ.

---

## 2) Single model for top/front/side: treat view as a conditioning signal
Training one net across three perspectives can work “naively” by mixing all frames, but it’s usually better to make the model *aware* of view identity.

### Minimal, effective approach: add a view one-hot input
Add 3 constant channels (Top/Front/Side) to every pixel:
- `view_top`, `view_front`, `view_side` ∈ {0,1}

This is cheap and tends to reduce confusion when the same pixel patterns mean different things across views.

**Input channels then become**
- RGB: 3
- previous mask (one-hot): 3 (bg, A, B) — or 2 if you prefer A/B only
- view one-hot: 3  
Total: **9 channels** (or 8 if you drop bg from prev mask)

This keeps “single net” true while still acknowledging domain differences.

*(Optional upgrade later: conditional normalization per view, but try view one-hot first.)*

---

## 3) Option 1 temporal hint: how to feed the previous mask
### Use **soft** previous-mask channels, not hard argmax
At inference you will have probabilities, not ground truth. If you train on hard one-hot masks and infer on noisy argmax, you create a brittle discontinuity.

So: represent `prev_mask` as **three probability maps** (softmax outputs) during inference, and during training you can use either:
- ground-truth one-hot (clean) **or**
- a lightly blurred one-hot (to mimic uncertainty) **or**
- a mix of GT and predicted (scheduled sampling; below)

This makes the network learn “how to use a hint” rather than “how to trust a perfect oracle”.

---

## 4) The key pitfall: teacher forcing mismatch → scheduled sampling
If you always feed the ground-truth previous mask during training, but at test time you feed the model’s own previous prediction, small errors can accumulate and the model may over-rely on a perfect prior.

### Practical training strategy (works well and is simple)
Train in phases:

**Phase 1 (stabilize segmentation):**  
- Feed **GT prev mask** always.
- Train until per-view IoU/Dice plateaus.

**Phase 2 (robustify to self-feedback):**  
- For each training sample (t-1, t), choose prev mask source:
  - with prob `p`: use **predicted prev mask** from running the model on frame t-1
  - with prob `1-p`: use **GT prev mask**
- Increase `p` gradually (e.g. 0 → 0.5 → 0.8)

This is the standard “scheduled sampling” idea, and it directly attacks error accumulation.

**Compute note:** doing an extra forward pass for t-1 costs something, but on a 4090 laptop it’s still typically fine if you keep resolution reasonable. If it’s too slow, you can approximate by using “corrupted GT prev masks” instead of predicted ones.

---

## 5) Data sampling: train on pairs (t-1, t), not full sequences
You don’t need long unrolls to get most of the benefit.

Your dataset item can be:
- current RGB frame `I_t`
- previous RGB frame optional (not necessary if you only want prev mask)
- previous mask `M_{t-1}`
- current mask `M_t`
- view id `v`

Train on random frame pairs within each clip, with special attention to:
- high-motion segments
- interaction segments
- occlusion-heavy segments (front/side)

A simple way to ensure this is to **oversample** frames where overlap/occlusion occurs (you can detect this from masks: e.g., small visible area for one mouse, close centroids, bounding box overlap, etc.).

---

## 6) Architecture choice: U-Net is fine; just ensure it handles 9-channel input cleanly
A standard U-Net works. Two practical points:

1) **Normalization:** use GroupNorm/InstanceNorm (or freeze BN) if batch size is small. Small-batch BN across mixed views can be noisy.

2) **Encoder initialization:** if you use an ImageNet-pretrained encoder (expects 3 channels), you’ll need to adapt the first conv to 9 channels. Easy ways:
- initialize extra channels with zeros
- or copy/average RGB weights into extra channels

This keeps training stable on limited compute.

---

## 7) Losses: keep it boring, but weight identity classes appropriately
For 3-class segmentation (bg/A/B):
- Cross-entropy + Dice is enough.
- Give background a lower weight if it dominates.

Because fur colors differ strongly, identity is mostly appearance-driven; the prev-mask channels mainly help during occlusion and reduce ID jitter.

---

## 8) Expected behavior under occlusion
With **modal labels**, the model should learn:
- when mouse B is hidden behind mouse A, those pixels belong to A (or bg), not B
- but when a small visible fragment of B exists, it should be detected and labeled as B

The previous mask helps the network keep “belief” about where B likely is, but the ground truth should still win where visibility says otherwise.

If you truly want “partially occluded but not visible” pixels to still be labeled as the occluded mouse (amodal), then the model is being asked to hallucinate unseen regions. It can still work given perfect simulation labels, but you’ll be training a different task.

## 9) Minimal inference loop

1. Set `prev_mask = all-background` (or use GT if available for the first frame).
2. For each frame `I_t`:

   - set `prev_mask = P_t` (soft probabilities recommended)

This gives you temporally coherent masks without an explicit tracker. (You can still add a tracker later if you want trajectory-level outputs.)

```python
def decode_mask(mask_arr: np.ndarray) -> np.ndarray:
    """
    Returns HxW uint8 with labels in {0,1,2}.
    If your simulator stores masks as grayscale indices, this is already correct.
    If it stores RGB colors, implement mapping here.
    """
    if mask_arr.ndim == 2:
        # Already single channel (good)
        out = mask_arr.astype(np.uint8)
        return out
    if mask_arr.ndim == 3 and mask_arr.shape[2] in (3, 4):
        # Possibly RGB-coded. Default behavior: convert to gray and hope values are 0/1/2.
        # If not true, replace this with a color->label mapping.
        if mask_arr.shape[2] == 4:
            mask_arr = mask_arr[:, :, :3]
        # OpenCV loads BGR; if this is from cv2.imread, it's BGR. If passed in already, might be RGB.
        # We'll treat as BGR here.
        gray = cv2.cvtColor(mask_arr, cv2.COLOR_BGR2GRAY)
        out = gray.astype(np.uint8)
        return out
    raise ValueError(f"Unsupported mask shape: {mask_arr.shape}")

def to_one_hot(mask_hw: torch.Tensor, num_classes: int = 3) -> torch.Tensor:
    """
    mask_hw: (H,W) long
    returns: (C,H,W) float
    """
    oh = F.one_hot(mask_hw.long(), num_classes=num_classes).permute(2, 0, 1).float()
    return oh

def view_one_hot(view_idx: int, h: int, w: int) -> torch.Tensor:
    """
    returns: (3,H,W) float constant planes
    """
    v = torch.zeros(3, h, w, dtype=torch.float32)
    v[view_idx, :, :] = 1.0
    return v

def resize_pair(
    img: np.ndarray,
    mask: np.ndarray,
    out_hw: Tuple[int, int],
    mode: str = "stretch",
) -> Tuple[np.ndarray, np.ndarray]:
    """
    out_hw: (H, W)
    mode:
      - "stretch": direct resize
      - "letterbox": keep aspect, pad with zeros for img and 0 for mask
    """
    out_h, out_w = out_hw
    h, w = img.shape[:2]

    if mode == "stretch":
        img_r = cv2.resize(img, (out_w, out_h), interpolation=cv2.INTER_LINEAR)
        mask_r = cv2.resize(mask, (out_w, out_h), interpolation=cv2.INTER_NEAREST)
        return img_r, mask_r

    if mode == "letterbox":
        scale = min(out_w / w, out_h / h)
        nw, nh = int(round(w * scale)), int(round(h * scale))
        img_s = cv2.resize(img, (nw, nh), interpolation=cv2.INTER_LINEAR)
        mask_s = cv2.resize(mask, (nw, nh), interpolation=cv2.INTER_NEAREST)

        canvas_img = np.zeros((out_h, out_w, 3), dtype=img.dtype)
        canvas_mask = np.zeros((out_h, out_w), dtype=mask.dtype)

        x0 = (out_w - nw) // 2
        y0 = (out_h - nh) // 2
        canvas_img[y0:y0 + nh, x0:x0 + nw] = img_s
        canvas_mask[y0:y0 + nh, x0:x0 + nw] = mask_s
        return canvas_img, canvas_mask

    raise ValueError(f"Unknown resize mode: {mode}")

def simple_augment(img: np.ndarray, mask: np.ndarray, p_flip: float = 0.5) -> Tuple[np.ndarray, np.ndarray]:
    # Minimal safe augmentation (works without extra deps): horizontal flip + small brightness jitter.
    if random.random() < p_flip:
        img = img[:, ::-1, :].copy()
        mask = mask[:, ::-1].copy()
    if random.random() < 0.3:
        alpha = random.uniform(0.9, 1.1)  # contrast
        beta = random.uniform(-10, 10)    # brightness
        img = np.clip(alpha * img + beta, 0, 255).astype(np.uint8)
    return img, mask

class VideoReader:
    def __init__(self, path: Path, grayscale: bool = False):
        self.path = path
        self.grayscale = grayscale
        self.cap = cv2.VideoCapture(str(path))
        if not self.cap.isOpened():
            raise FileNotFoundError(f"Cannot open video: {path}")
        self._len = int(self.cap.get(cv2.CAP_PROP_FRAME_COUNT))

    def __len__(self) -> int:
        return self._len

    def read_frame(self, idx: int) -> np.ndarray:
        if idx < 0 or idx >= self._len:
            raise IndexError(idx)
        self.cap.set(cv2.CAP_PROP_POS_FRAMES, idx)
        ok, frame = self.cap.read()
        if not ok or frame is None:
            raise RuntimeError(f"Failed reading frame {idx} from {self.path}")
        if self.grayscale:
            if frame.ndim == 3:
                frame = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
            return frame
        frame = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
        return frame

    def close(self):
        if self.cap is not None:
            self.cap.release()
            self.cap = None

@dataclass
class SampleInfo:
    sample_id: str
    view: str
    rgb_path: Path
    mask_path: Path
    camera_path: Optional[Path] = None

def load_manifest(path: Path) -> List[SampleInfo]:
    items: List[SampleInfo] = []
    with path.open("r", newline="") as f:
        reader = csv.DictReader(f)
        required = {"sample_id", "view", "rgb_path", "mask_path"}
        missing = required - set(reader.fieldnames or [])
        if missing:
            raise ValueError(f"manifest.csv missing columns: {missing}")
        for row in reader:
            view = row["view"].strip().lower()
            if view not in VIEWS:
                raise ValueError(f"Unknown view '{view}' for sample_id={row['sample_id']}")
            cam = row.get("camera_path", "").strip()
            items.append(
                SampleInfo(
                    sample_id=row["sample_id"].strip(),
                    view=view,
                    rgb_path=Path(row["rgb_path"].strip()),
                    mask_path=Path(row["mask_path"].strip()),
                    camera_path=Path(cam) if cam else None,
                )
            )
    return items

def load_ethogram(path: Path) -> Dict[str, List[Tuple[int, int, str]]]:
    """
    returns:
      dict sample_id -> list of (start_frame, end_frame, behavior)
    """
    by_vid: Dict[str, List[Tuple[int, int, str]]] = {}
    with path.open("r", newline="") as f:
        reader = csv.DictReader(f)
        required = {"sample_id", "start_frame", "end_frame", "behavior"}
        missing = required - set(reader.fieldnames or [])
        if missing:
            raise ValueError(f"ethogram.csv missing columns: {missing}")
        for row in reader:
            sid = row["sample_id"].strip()
            s = int(row["start_frame"])
            e = int(row["end_frame"])
            beh = row["behavior"].strip()
            by_vid.setdefault(sid, []).append((s, e, beh))
    return by_vid

def build_behavior_array(num_frames: int, intervals: List[Tuple[int, int, str]]) -> List[str]:
    beh = [""] * num_frames
    for s, e, b in intervals:
        s = max(0, s)
        e = min(num_frames - 1, e)
        for i in range(s, e + 1):
            beh[i] = b
    return beh

class MicePairDataset(Dataset):
    """
    Provides samples at time t with previous mask at t-1:
      x = concat( RGB_t, prev_mask_onehot(t-1), view_onehot )
      y = mask(t)

    Supports oversampling weights by behavior class via WeightedRandomSampler.
    """
    def __init__(
        self,
        samples: List[SampleInfo],
        out_hw: Tuple[int, int] = (512, 512),
        resize_mode: str = "stretch",
        augment: bool = True,
        ethogram: Optional[Dict[str, List[Tuple[int, int, str]]]] = None,
        oversample_behaviors: Optional[List[str]] = None,
        oversample_factor: float = 3.0,
        min_t: int = 1,
    ):
        self.samples = samples
        self.out_hw = out_hw
        self.resize_mode = resize_mode
        self.augment = augment

        self.ethogram = ethogram or {}
        self.oversample_behaviors = set([b.strip() for b in (oversample_behaviors or []) if b.strip()])
        self.oversample_factor = float(oversample_factor)

        # Build index: list of (sample_idx, t)
        self.index: List[Tuple[int, int]] = []
        self.weights: List[float] = []

        for si, info in enumerate(samples):
            n = self._num_frames(info)
            intervals = self.ethogram.get(info.sample_id, [])
            beh_arr = build_behavior_array(n, intervals) if intervals else [""] * n

            for t in range(min_t, n):
                self.index.append((si, t))
                w = 1.0
                if self.oversample_behaviors and beh_arr[t] in self.oversample_behaviors:
                    w = self.oversample_factor
                self.weights.append(w)

    def _num_frames(self, info: SampleInfo) -> int:
        if info.rgb_path.is_dir():
            return len(list_frames(info.rgb_path))
        vr = VideoReader(info.rgb_path, grayscale=False)
        n = len(vr)
        vr.close()
        return n

    def __len__(self) -> int:
        return len(self.index)

    def _read_rgb(self, info: SampleInfo, t: int) -> np.ndarray:
        if info.rgb_path.is_dir():
            frames = list_frames(info.rgb_path)
            return read_image(frames[t])
        vr = VideoReader(info.rgb_path, grayscale=False)
        img = vr.read_frame(t)
        vr.close()
        return img

    def _read_mask(self, info: SampleInfo, t: int) -> np.ndarray:
        if info.mask_path.is_dir():
            frames = list_frames(info.mask_path)
            return read_mask(frames[t])
        vr = VideoReader(info.mask_path, grayscale=True)
        m = vr.read_frame(t)
        vr.close()
        return decode_mask(m)

    def __getitem__(self, idx: int):
        si, t = self.index[idx]
        info = self.samples[si]
        view_idx = VIEWS[info.view]

        img_t = self._read_rgb(info, t)
        mask_t = self._read_mask(info, t)
        prev_mask = self._read_mask(info, t - 1)

        # Resize
        img_t, mask_t = resize_pair(img_t, mask_t, self.out_hw, mode=self.resize_mode)
        _, prev_mask = resize_pair(img_t, prev_mask, self.out_hw, mode=self.resize_mode)  # img ignored; uses size

        if self.augment:
            img_t, mask_t = simple_augment(img_t, mask_t)
            # Apply same flip/jitter to prev_mask? Flip matters, jitter doesn't.
            # Our simple_augment flips both img and mask; we must flip prev_mask the same way.
            # We replicate flip decision by re-doing the flip with same random seed is messy;
            # instead, do deterministic: recompute flip once here.
            # So: implement flip explicitly.
            # (We'll keep minimal, correct behavior: do a second flip draw and accept slight mismatch is bad.)
            # Better: implement flip ourselves deterministically:
            # For now, we handle it correctly:
            # We'll implement deterministic flip here instead of above.
        # Correct deterministic augmentation (override previous)
        if self.augment:
            do_flip = (random.random() < 0.5)
            if do_flip:
                img_t = img_t[:, ::-1, :].copy()
                mask_t = mask_t[:, ::-1].copy()
                prev_mask = prev_mask[:, ::-1].copy()
            if random.random() < 0.3:
                alpha = random.uniform(0.9, 1.1)
                beta = random.uniform(-10, 10)
                img_t = np.clip(alpha * img_t + beta, 0, 255).astype(np.uint8)

        # To tensors
        img_t = torch.from_numpy(img_t).permute(2, 0, 1).float() / 255.0  # (3,H,W)
        mask_t = torch.from_numpy(mask_t.astype(np.int64))                 # (H,W)
        prev_mask = torch.from_numpy(prev_mask.astype(np.int64))           # (H,W)

        prev_oh = to_one_hot(prev_mask, 3)                                 # (3,H,W)
        v_oh = view_one_hot(view_idx, img_t.shape[1], img_t.shape[2])       # (3,H,W)

        x = torch.cat([img_t, prev_oh, v_oh], dim=0)                        # (9,H,W)
        y = mask_t                                                         # (H,W)
        return x, y, view_idx

# ----------------------------
# Model: U-Net (GroupNorm)
# ----------------------------

def gn(num_channels: int, num_groups: int = 32) -> nn.GroupNorm:
    g = min(num_groups, num_channels)
    while num_channels % g != 0 and g > 1:
        g -= 1
    return nn.GroupNorm(g, num_channels)

class DoubleConv(nn.Module):
    def __init__(self, in_ch: int, out_ch: int):
        super().__init__()
        self.net = nn.Sequential(
            nn.Conv2d(in_ch, out_ch, 3, padding=1, bias=False),
            gn(out_ch),
            nn.ReLU(inplace=True),
            nn.Conv2d(out_ch, out_ch, 3, padding=1, bias=False),
            gn(out_ch),
            nn.ReLU(inplace=True),
        )

    def forward(self, x):
        return self.net(x)

class UNet(nn.Module):
    def __init__(self, in_channels: int = 9, num_classes: int = 3, base: int = 32):
        super().__init__()
        self.inc = DoubleConv(in_channels, base)
        self.down1 = nn.Sequential(nn.MaxPool2d(2), DoubleConv(base, base * 2))
        self.down2 = nn.Sequential(nn.MaxPool2d(2), DoubleConv(base * 2, base * 4))
        self.down3 = nn.Sequential(nn.MaxPool2d(2), DoubleConv(base * 4, base * 8))
        self.down4 = nn.Sequential(nn.MaxPool2d(2), DoubleConv(base * 8, base * 16))

        self.up1 = nn.ConvTranspose2d(base * 16, base * 8, 2, stride=2)
        self.conv1 = DoubleConv(base * 16, base * 8)

        self.up2 = nn.ConvTranspose2d(base * 8, base * 4, 2, stride=2)
        self.conv2 = DoubleConv(base * 8, base * 4)

        self.up3 = nn.ConvTranspose2d(base * 4, base * 2, 2, stride=2)
        self.conv3 = DoubleConv(base * 4, base * 2)

        self.up4 = nn.ConvTranspose2d(base * 2, base, 2, stride=2)
        self.conv4 = DoubleConv(base * 2, base)

        self.outc = nn.Conv2d(base, num_classes, 1)

    @staticmethod
    def _pad_to(x, ref):
        # pad x to match spatial dims of ref
        diff_y = ref.size(2) - x.size(2)
        diff_x = ref.size(3) - x.size(3)
        return F.pad(x, [diff_x // 2, diff_x - diff_x // 2, diff_y // 2, diff_y - diff_y // 2])

    def forward(self, x):
        x1 = self.inc(x)
        x2 = self.down1(x1)
        x3 = self.down2(x2)
        x4 = self.down3(x3)
        x5 = self.down4(x4)

        u1 = self.up1(x5)
        u1 = self._pad_to(u1, x4)
        u1 = torch.cat([x4, u1], dim=1)
        u1 = self.conv1(u1)

        u2 = self.up2(u1)
        u2 = self._pad_to(u2, x3)
        u2 = torch.cat([x3, u2], dim=1)
        u2 = self.conv2(u2)

        u3 = self.up3(u2)
        u3 = self._pad_to(u3, x2)
        u3 = torch.cat([x2, u3], dim=1)
        u3 = self.conv3(u3)

        u4 = self.up4(u3)
        u4 = self._pad_to(u4, x1)
        u4 = torch.cat([x1, u4], dim=1)
        u4 = self.conv4(u4)

        logits = self.outc(u4)
        return logits

# ----------------------------
# Losses / Metrics
# ----------------------------

class DiceLoss(nn.Module):
    def __init__(self, num_classes: int = 3, smooth: float = 1.0, ignore_bg: bool = True):
        super().__init__()
        self.num_classes = num_classes
        self.smooth = smooth
        self.ignore_bg = ignore_bg

    def forward(self, logits: torch.Tensor, target: torch.Tensor) -> torch.Tensor:
        """
        logits: (B,C,H,W)
        target: (B,H,W) long
        """
        probs = torch.softmax(logits, dim=1)
        target_oh = F.one_hot(target, num_classes=self.num_classes).permute(0, 3, 1, 2).float()

        if self.ignore_bg:
            probs = probs[:, 1:, :, :]
            target_oh = target_oh[:, 1:, :, :]

        dims = (0, 2, 3)
        inter = torch.sum(probs * target_oh, dims)
        denom = torch.sum(probs + target_oh, dims)
        dice = (2 * inter + self.smooth) / (denom + self.smooth)
        return 1.0 - dice.mean()

@torch.no_grad()
def mean_iou(pred: torch.Tensor, target: torch.Tensor, num_classes: int = 3) -> float:
    """
    pred: (B,H,W) long
    target: (B,H,W) long
    """
    ious = []
    for c in range(num_classes):
        p = (pred == c)
        t = (target == c)
        inter = (p & t).sum().item()
        union = (p | t).sum().item()
        if union == 0:
            continue
        ious.append(inter / union)
    return float(np.mean(ious)) if ious else 0.0

@dataclass
class TrainConfig:
    manifest: Path
    ethogram: Optional[Path]
    out_dir: Path
    image_size: int = 512
    resize_mode: str = "stretch"
    batch_size: int = 6
    num_workers: int = 4
    lr: float = 2e-4
    weight_decay: float = 1e-4
    epochs: int = 40
    seed: int = 123
    val_split: float = 0.2
    oversample_behaviors: Optional[List[str]] = None
    oversample_factor: float = 3.0
    device: str = "cuda"
    base_channels: int = 32
    ce_bg_weight: float = 0.3   # down-weight background in CE
    amp: bool = True
    grad_accum: int = 1
    save_every: int = 1

def split_by_sample_id(items: List[SampleInfo], val_frac: float, seed: int) -> Tuple[List[SampleInfo], List[SampleInfo]]:
    rng = random.Random(seed)
    ids = sorted({x.sample_id for x in items})
    rng.shuffle(ids)
    n_val = max(1, int(round(len(ids) * val_frac)))
    val_ids = set(ids[:n_val])
    train = [x for x in items if x.sample_id not in val_ids]
    val = [x for x in items if x.sample_id in val_ids]
    return train, val

def build_class_weights(bg_weight: float = 0.3) -> torch.Tensor:
    # weights for CE: [bg, mouseA, mouseB]
    return torch.tensor([bg_weight, 1.0, 1.0], dtype=torch.float32)

def save_checkpoint(path: Path, model: nn.Module, optim: torch.optim.Optimizer, scaler: GradScaler, epoch: int) -> None:
    ensure_dir(path.parent)
    torch.save({
        "model": model.state_dict(),
        "optim": optim.state_dict(),
        "scaler": scaler.state_dict(),
        "epoch": epoch,
    }, path)

def load_checkpoint(path: Path, model: nn.Module, optim: Optional[torch.optim.Optimizer] = None, scaler: Optional[GradScaler] = None):
    ckpt = torch.load(path, map_location="cpu")
    model.load_state_dict(ckpt["model"])
    if optim is not None and "optim" in ckpt:
        optim.load_state_dict(ckpt["optim"])
    if scaler is not None and "scaler" in ckpt:
        scaler.load_state_dict(ckpt["scaler"])
    return int(ckpt.get("epoch", 0))

def train(cfg: TrainConfig) -> None:
    set_seed(cfg.seed)
    ensure_dir(cfg.out_dir)

    items = load_manifest(cfg.manifest)
    eth = load_ethogram(cfg.ethogram) if cfg.ethogram else None

    train_items, val_items = split_by_sample_id(items, cfg.val_split, cfg.seed)

    out_hw = (cfg.image_size, cfg.image_size)
    train_ds = MicePairDataset(
        train_items,
        out_hw=out_hw,
        resize_mode=cfg.resize_mode,
        augment=True,
        ethogram=eth,
        oversample_behaviors=cfg.oversample_behaviors,
        oversample_factor=cfg.oversample_factor,
        min_t=1,
    )
    val_ds = MicePairDataset(
        val_items,
        out_hw=out_hw,
        resize_mode=cfg.resize_mode,
        augment=False,
        ethogram=None,
        oversample_behaviors=None,
        oversample_factor=1.0,
        min_t=1,
    )

    sampler = None
    shuffle = True
    if cfg.oversample_behaviors:
        sampler = WeightedRandomSampler(train_ds.weights, num_samples=len(train_ds.weights), replacement=True)
        shuffle = False

    train_loader = DataLoader(
        train_ds,
        batch_size=cfg.batch_size,
        shuffle=shuffle,
        sampler=sampler,
        num_workers=cfg.num_workers,
        pin_memory=True,
        drop_last=True,
    )
    val_loader = DataLoader(
        val_ds,
        batch_size=max(1, cfg.batch_size // 2),
        shuffle=False,
        num_workers=cfg.num_workers,
        pin_memory=True,
        drop_last=False,
    )

    device = torch.device(cfg.device if torch.cuda.is_available() else "cpu")
    model = UNet(in_channels=9, num_classes=3, base=cfg.base_channels).to(device)

    ce_w = build_class_weights(cfg.ce_bg_weight).to(device)
    ce = nn.CrossEntropyLoss(weight=ce_w)
    dice = DiceLoss(num_classes=3, ignore_bg=True)

    optim = torch.optim.AdamW(model.parameters(), lr=cfg.lr, weight_decay=cfg.weight_decay)
    scaler = GradScaler(enabled=(cfg.amp and device.type == "cuda"))

    best_val = -1.0

    for epoch in range(1, cfg.epochs + 1):
        model.train()
        running_loss = 0.0
        running_iou = 0.0
        n_batches = 0

        optim.zero_grad(set_to_none=True)

        for step, (x, y, view_idx) in enumerate(train_loader, start=1):
            x = x.to(device, non_blocking=True)
            y = y.to(device, non_blocking=True)

            with autocast(enabled=(cfg.amp and device.type == "cuda")):
                logits = model(x)
                loss = ce(logits, y) + dice(logits, y)

            scaler.scale(loss / cfg.grad_accum).backward()

            if step % cfg.grad_accum == 0:
                scaler.step(optim)
                scaler.update()
                optim.zero_grad(set_to_none=True)

            running_loss += loss.item()
            with torch.no_grad():
                pred = torch.argmax(logits, dim=1)
                running_iou += mean_iou(pred, y, num_classes=3)
            n_batches += 1

        train_loss = running_loss / max(1, n_batches)
        train_iou = running_iou / max(1, n_batches)

        # Validation
        model.eval()
        val_loss_sum = 0.0
        val_iou_sum = 0.0
        val_batches = 0
        # also per-view IOU
        per_view = {0: [], 1: [], 2: []}

        with torch.no_grad():
            for x, y, view_idx in val_loader:
                x = x.to(device, non_blocking=True)
                y = y.to(device, non_blocking=True)
                view_idx = view_idx.cpu().numpy().tolist()

                with autocast(enabled=(cfg.amp and device.type == "cuda")):
                    logits = model(x)
                    loss = ce(logits, y) + dice(logits, y)

                pred = torch.argmax(logits, dim=1)
                iou = mean_iou(pred, y, num_classes=3)

                val_loss_sum += loss.item()
                val_iou_sum += iou
                val_batches += 1

                # batch-level per-view assignment
                for bi, v in enumerate(view_idx):
                    pv = pred[bi:bi+1]
                    yv = y[bi:bi+1]
                    per_view[int(v)].append(mean_iou(pv, yv, num_classes=3))

        val_loss = val_loss_sum / max(1, val_batches)
        val_iou = val_iou_sum / max(1, val_batches)

        pv_str = " | ".join(
            f"{IDX_TO_VIEW[k]}:{(sum(vals)/len(vals)) if vals else 0.0:.3f}"
            for k, vals in per_view.items()
        )

        print(
            f"Epoch {epoch:03d}/{cfg.epochs} "
            f"train_loss={train_loss:.4f} train_mIoU={train_iou:.3f} | "
            f"val_loss={val_loss:.4f} val_mIoU={val_iou:.3f} | {pv_str}"
        )

        # Save checkpoints
        if epoch % cfg.save_every == 0:
            save_checkpoint(cfg.out_dir / f"ckpt_epoch_{epoch:03d}.pt", model, optim, scaler, epoch)

        if val_iou > best_val:
            best_val = val_iou
            save_checkpoint(cfg.out_dir / "ckpt_best.pt", model, optim, scaler, epoch)

def colorize_mask(mask_hw: np.ndarray) -> np.ndarray:
    """
    mask: HxW uint8 with {0,1,2}
    returns: HxWx3 uint8 RGB
    """
    h, w = mask_hw.shape
    out = np.zeros((h, w, 3), dtype=np.uint8)
    # Background: black
    out[mask_hw == 1] = (255, 0, 0)   # mouse A: red
    out[mask_hw == 2] = (0, 255, 0)   # mouse B: green
    return out

@torch.no_grad()
def predict_sequence(
    model: nn.Module,
    rgb_source: Path,
    view: str,
    out_dir: Path,
    image_size: int = 512,
    resize_mode: str = "stretch",
    device: str = "cuda",
    save_overlay: bool = True,
    alpha: float = 0.4,
):
    ensure_dir(out_dir)
    device_t = torch.device(device if torch.cuda.is_available() else "cpu")
    model.eval().to(device_t)

    view_idx = VIEWS[view.lower()]

    # Prepare frame iterator
    if rgb_source.is_dir():
        frames = list_frames(rgb_source)
        n = len(frames)
        def get_rgb(i: int) -> np.ndarray:
            return read_image(frames[i])
    else:
        vr = VideoReader(rgb_source, grayscale=False)
        n = len(vr)
        def get_rgb(i: int) -> np.ndarray:
            return vr.read_frame(i)

    out_hw = (image_size, image_size)

    # prev mask probs: start as all background
    prev_probs = torch.zeros(1, 3, image_size, image_size, device=device_t, dtype=torch.float32)
    prev_probs[:, 0, :, :] = 1.0

    for t in range(n):
        img = get_rgb(t)
        # resize image to match prev_probs
        img_r = cv2.resize(img, (image_size, image_size), interpolation=cv2.INTER_LINEAR) if resize_mode == "stretch" else resize_pair(img, np.zeros(img.shape[:2], np.uint8), out_hw, resize_mode)[0]
        img_t = torch.from_numpy(img_r).permute(2, 0, 1).float().unsqueeze(0) / 255.0  # (1,3,H,W)

        v_oh = view_one_hot(view_idx, image_size, image_size).unsqueeze(0)              # (1,3,H,W)
        x = torch.cat([img_t.to(device_t), prev_probs, v_oh.to(device_t)], dim=1)       # (1,9,H,W)

        logits = model(x)
        probs = torch.softmax(logits, dim=1)
        pred = torch.argmax(probs, dim=1).squeeze(0).detach().cpu().numpy().astype(np.uint8)

        # update prev probs autoregressively (soft, recommended)
        prev_probs = probs.detach()

        # save outputs
        mask_path = out_dir / f"mask_{t:06d}.png"
        cv2.imwrite(str(mask_path), pred)  # single-channel

        if save_overlay:
            col = colorize_mask(pred)
            # overlay on resized image
            overlay = (1 - alpha) * img_r.astype(np.float32) + alpha * col.astype(np.float32)
            overlay = np.clip(overlay, 0, 255).astype(np.uint8)
            ov_path = out_dir / f"overlay_{t:06d}.png"
            cv2.imwrite(str(ov_path), cv2.cvtColor(overlay, cv2.COLOR_RGB2BGR))

    if not rgb_source.is_dir():
        vr.close()

# Frame extraction (recommended)

def extract_frames(
    rgb_video: Path,
    mask_video: Optional[Path],
    out_rgb_dir: Path,
    out_mask_dir: Optional[Path],
    every: int = 1,
):
    ensure_dir(out_rgb_dir)
    if out_mask_dir is not None:
        ensure_dir(out_mask_dir)

    vr = VideoReader(rgb_video, grayscale=False)
    n = len(vr)

    mr = None
    if mask_video is not None:
        mr = VideoReader(mask_video, grayscale=True)
        if len(mr) != n:
            print(f"Warning: rgb frames={n}, mask frames={len(mr)} (continuing with min length)")
            n = min(n, len(mr))

    for i in range(0, n, every):
        img = vr.read_frame(i)  # RGB
        outp = out_rgb_dir / f"{i:06d}.png"
        cv2.imwrite(str(outp), cv2.cvtColor(img, cv2.COLOR_RGB2BGR))

        if mr is not None and out_mask_dir is not None:
            m = mr.read_frame(i)
            m = decode_mask(m)
            outm = out_mask_dir / f"{i:06d}.png"
            cv2.imwrite(str(outm), m)

    vr.close()
    if mr is not None:
        mr.close()
```

## 2) Update `predict_sequence()` signature + body

Replace your existing `predict_sequence(...)` with this version:

```python
@torch.no_grad()
def predict_sequence(
    model: nn.Module,
    rgb_source: Path,
    view: str,
    out_dir: Path,
    image_size: int = 512,
    resize_mode: str = "stretch",
    device: str = "cuda",
    save_overlay: bool = True,
    alpha: float = 0.4,
    out_mask_video: Optional[Path] = None,
    mouse0_bgr: Tuple[int, int, int] = (25, 35, 40),
    mouse1_bgr: Tuple[int, int, int] = (40, 35, 25),
    fps: Optional[float] = None,
):
    ensure_dir(out_dir)
    device_t = torch.device(device if torch.cuda.is_available() else "cpu")
    model.eval().to(device_t)

    view_idx = VIEWS[view.lower()]

    # Frame iterator + FPS (if source is a video)
    src_is_dir = rgb_source.is_dir()
    if src_is_dir:
        frames = list_frames(rgb_source)
        n = len(frames)
        def get_rgb(i: int) -> np.ndarray:
            return read_image(frames[i])
        src_fps = float(fps) if fps is not None else 30.0
    else:
        cap = cv2.VideoCapture(str(rgb_source))
        if not cap.isOpened():
            raise FileNotFoundError(f"Cannot open video: {rgb_source}")
        n = int(cap.get(cv2.CAP_PROP_FRAME_COUNT))
        cap_fps = float(cap.get(cv2.CAP_PROP_FPS) or 0.0)
        src_fps = float(fps) if fps is not None else (cap_fps if cap_fps > 1e-3 else 30.0)

        def get_rgb(i: int) -> np.ndarray:
            cap.set(cv2.CAP_PROP_POS_FRAMES, i)
            ok, frame_bgr = cap.read()
            if not ok or frame_bgr is None:
                raise RuntimeError(f"Failed reading frame {i} from {rgb_source}")
            return cv2.cvtColor(frame_bgr, cv2.COLOR_BGR2RGB)

    out_hw = (image_size, image_size)

    # prev mask probs at model resolution: start as all-background
    prev_probs = torch.zeros(1, 3, image_size, image_size, device=device_t, dtype=torch.float32)
    prev_probs[:, 0, :, :] = 1.0

    # Optional video writer (initialized lazily once we know original frame size)
    vw = None
    video_size = None  # (W,H)

    for t in range(n):
        img_orig_rgb = get_rgb(t)
        H0, W0 = img_orig_rgb.shape[:2]

        # Initialize video writer once we know size
        if out_mask_video is not None and vw is None:
            ensure_dir(out_mask_video.parent)
            video_size = (W0, H0)
            vw = cv2.VideoWriter(
                str(out_mask_video),
                fourcc_for_path(out_mask_video),
                src_fps,
                video_size,
                True,
            )
            if not vw.isOpened():
                raise RuntimeError(f"Failed to open VideoWriter for {out_mask_video}")

        # Prepare model input: resize image to model resolution
        if resize_mode == "stretch":
            img_model_rgb = cv2.resize(img_orig_rgb, (image_size, image_size), interpolation=cv2.INTER_LINEAR)
        else:
            img_model_rgb, _ = resize_pair(
                img_orig_rgb,
                np.zeros((H0, W0), np.uint8),
                out_hw,
                mode=resize_mode
            )

        img_t = torch.from_numpy(img_model_rgb).permute(2, 0, 1).float().unsqueeze(0) / 255.0  # (1,3,H,W)
        v_oh = view_one_hot(view_idx, image_size, image_size).unsqueeze(0)                        # (1,3,H,W)
        x = torch.cat([img_t.to(device_t), prev_probs, v_oh.to(device_t)], dim=1)                 # (1,9,H,W)

        logits = model(x)
        probs = torch.softmax(logits, dim=1)
        pred_model = torch.argmax(probs, dim=1).squeeze(0).detach().cpu().numpy().astype(np.uint8)

        # Autoregressive update (soft)
        prev_probs = probs.detach()

        # Save mask frame (at model resolution)
        mask_path = out_dir / f"mask_{t:06d}.png"
        cv2.imwrite(str(mask_path), pred_model)

        # Optional overlay PNG (at model resolution)
        if save_overlay:
            col = colorize_mask(pred_model)
            overlay = (1 - alpha) * img_model_rgb.astype(np.float32) + alpha * col.astype(np.float32)
            overlay = np.clip(overlay, 0, 255).astype(np.uint8)
            ov_path = out_dir / f"overlay_{t:06d}.png"
            cv2.imwrite(str(ov_path), cv2.cvtColor(overlay, cv2.COLOR_RGB2BGR))

        # Optional composited video (at original resolution)
        if vw is not None:
            # resize predicted mask back to original size (nearest)
            pred_orig = cv2.resize(pred_model, (W0, H0), interpolation=cv2.INTER_NEAREST)

            # composite in BGR (VideoWriter expects BGR)
            out_bgr = cv2.cvtColor(img_orig_rgb, cv2.COLOR_RGB2BGR)
            out_bgr[pred_orig == 1] = mouse0_bgr
            out_bgr[pred_orig == 2] = mouse1_bgr
            vw.write(out_bgr)

    if not src_is_dir:
        cap.release()
    if vw is not None:
        vw.release()
```

```python
# ----------------------------
# Camera.json resolution assertions

_CAMCACHE: Dict[str, dict] = {}

def load_cameras_json(path: Path) -> dict:
    key = str(path.resolve())
    if key in _CAMCACHE:
        return _CAMCACHE[key]
    with path.open("r") as f:
        data = json.load(f)
    if "cameras" not in data or not isinstance(data["cameras"], list):
        raise ValueError(f"{path} does not look like a cameras.json (missing 'cameras' list)")
    _CAMCACHE[key] = data
    return data

def camera_resolution_for_view(cameras_data: dict, view: str) -> Tuple[int, int, str]:
    """
    Finds the camera whose name best matches view (top/front/side), e.g. cam1_top.
    Returns (width, height, camera_name).
    """
    view = view.lower().strip()
    if view not in VIEWS:
        raise ValueError(f"Unknown view '{view}' (expected one of {list(VIEWS.keys())})")

    cams = cameras_data["cameras"]
    # Prefer name ending with _{view}
    for cam in cams:
        name = str(cam.get("name", "")).lower()
        if name.endswith(f"_{view}"):
            return int(cam["width"]), int(cam["height"]), cam.get("name", "")
    # Fallback: contains _{view} anywhere
    for cam in cams:
        name = str(cam.get("name", "")).lower()
        if f"_{view}" in name:
            return int(cam["width"]), int(cam["height"]), cam.get("name", "")
    raise KeyError(f"No camera found in cameras.json matching view='{view}'")

def source_resolution_rgb(path: Path) -> Tuple[int, int]:
    """
    Returns (width, height) from either a frame folder or a video file (first frame).
    """
    if path.is_dir():
        frames = list_frames(path)
        if not frames:
            raise FileNotFoundError(f"No frames found in folder: {path}")
        img = read_image(frames[0])
        h, w = img.shape[:2]
        return (w, h)

    cap = cv2.VideoCapture(str(path))
    if not cap.isOpened():
        raise FileNotFoundError(f"Cannot open video: {path}")
    ok, frame_bgr = cap.read()
    cap.release()
    if not ok or frame_bgr is None:
        raise RuntimeError(f"Failed to read first frame from: {path}")
    h, w = frame_bgr.shape[:2]
    return (w, h)

def assert_matches_camera(view: str, rgb_path: Path, cameras_json_path: Path) -> None:
    cams = load_cameras_json(cameras_json_path)
    exp_w, exp_h, cam_name = camera_resolution_for_view(cams, view)
    act_w, act_h = source_resolution_rgb(rgb_path)
    if (act_w, act_h) != (exp_w, exp_h):
        raise ValueError(
            f"Resolution mismatch for view='{view}' camera='{cam_name}': "
            f"expected {exp_w}x{exp_h} from {cameras_json_path}, "
            f"but got {act_w}x{act_h} from {rgb_path}"
        )
```

# 1) Make GT label values configurable

### Add a label spec + remapper

```python
from dataclasses import dataclass

@dataclass(frozen=True)
class LabelSpec:
    bg: int = 0
    mouse0: int = 1
    mouse1: int = 2

def remap_gt_labels(mask_raw: np.ndarray, spec: LabelSpec) -> np.ndarray:
    """
    Map arbitrary raw label values -> canonical {0:bg, 1:mouse0, 2:mouse1}.
    Raises if unexpected labels appear.
    """
    if mask_raw.ndim != 2:
        raise ValueError(f"Expected 2D mask, got shape {mask_raw.shape}")

    out = np.full(mask_raw.shape, 255, dtype=np.uint8)  # sentinel
    out[mask_raw == spec.bg] = 0
    out[mask_raw == spec.mouse0] = 1
    out[mask_raw == spec.mouse1] = 2

    bad = np.unique(mask_raw[out == 255])
    if bad.size > 0:
        raise ValueError(
            f"Mask contains label values not in (bg={spec.bg}, mouse0={spec.mouse0}, mouse1={spec.mouse1}). "
            f"Unexpected values: {bad.tolist()}"
        )
    return out
```

Change `decode_mask(...)` to accept `label_spec` and call the remapper:

```python
def decode_mask(mask_arr: np.ndarray, label_spec: Optional[LabelSpec] = None) -> np.ndarray:
    """
    Returns HxW uint8 with canonical labels in {0,1,2}.
    """
    if mask_arr.ndim == 2:
        raw = mask_arr
    elif mask_arr.ndim == 3 and mask_arr.shape[2] in (3, 4):
        if mask_arr.shape[2] == 4:
            mask_arr = mask_arr[:, :, :3]
        gray = cv2.cvtColor(mask_arr, cv2.COLOR_BGR2GRAY)
        raw = gray
    else:
        raise ValueError(f"Unsupported mask shape: {mask_arr.shape}")

    raw = raw.astype(np.int64)
    if label_spec is None:
        # default: assume already 0/1/2
        return raw.astype(np.uint8)
    return remap_gt_labels(raw, label_spec)
```

# 2) Wire `cameras.json` into preprocessing (crop to cage ROI)

Your `cameras.json` provides:
- cage box (`origin`, `width`, `depth`, `height`) and
- per-camera `width/height`, intrinsics, and extrinsics (`fx,fy,cx,cy,R,t`) with names like `cam1_top`, `cam2_front`, `cam3_side`. fileciteturn5file0

There’s no distortion model in the file, so “wiring” intrinsics typically means:
- **assert resolution**, and
- **project the known cage 3D bounds into the image** → crop ROI.

### Add camera dataclass + cage projection crop

```python
@dataclass(frozen=True)
class CameraModel:
    name: str
    width: int
    height: int
    fx: float
    fy: float
    cx: float
    cy: float
    R: np.ndarray  # (3,3)
    t: np.ndarray  # (3,)

def camera_for_view(cameras_data: dict, view: str) -> CameraModel:
    view = view.lower().strip()
    cams = cameras_data["cameras"]

    def to_cam(cam: dict) -> CameraModel:
        return CameraModel(
            name=str(cam["name"]),
            width=int(cam["width"]),
            height=int(cam["height"]),
            fx=float(cam["fx"]),
            fy=float(cam["fy"]),
            cx=float(cam["cx"]),
            cy=float(cam["cy"]),
            R=np.array(cam["R"], dtype=np.float64),
            t=np.array(cam["t"], dtype=np.float64),
        )

    # prefer suffix match
    for cam in cams:
        if str(cam.get("name", "")).lower().endswith(f"_{view}"):
            return to_cam(cam)
    for cam in cams:
        if f"_{view}" in str(cam.get("name", "")).lower():
            return to_cam(cam)
    raise KeyError(f"No camera found matching view='{view}' in cameras.json")

def cage_corners_world(cameras_data: dict) -> np.ndarray:
    cage = cameras_data["cage"]
    ox, oy, oz = cage["origin"]
    w = float(cage["width"])
    d = float(cage["depth"])
    h = float(cage["height"])
    corners = []
    for dx in (0.0, w):
        for dy in (0.0, d):
            for dz in (0.0, h):
                corners.append([ox + dx, oy + dy, oz + dz])
    return np.array(corners, dtype=np.float64)  # (8,3)

def project_world_to_image(cam: CameraModel, Pw: np.ndarray) -> np.ndarray:
    """
    Pw: (N,3) world points.
    Returns uv: (N,2) image points (float).
    Uses pinhole: Pc = R*Pw + t; u = fx*x/z + cx; v = fy*y/z + cy
    """
    Pc = (cam.R @ Pw.T).T + cam.t.reshape(1, 3)  # (N,3)
    z = Pc[:, 2]
    eps = 1e-6
    valid = z > eps
    uv = np.full((Pw.shape[0], 2), np.nan, dtype=np.float64)
    uv[valid, 0] = cam.fx * (Pc[valid, 0] / z[valid]) + cam.cx
    uv[valid, 1] = cam.fy * (Pc[valid, 1] / z[valid]) + cam.cy
    return uv

def cage_crop_box_from_cameras_json(cameras_json_path: Path, view: str, margin: int = 10) -> Tuple[int,int,int,int]:
    cams_data = load_cameras_json(cameras_json_path)
    cam = camera_for_view(cams_data, view)
    Pw = cage_corners_world(cams_data)
    uv = project_world_to_image(cam, Pw)

    good = np.isfinite(uv[:, 0]) & np.isfinite(uv[:, 1])
    if good.sum() < 4:
        raise ValueError(f"Too few valid projected cage corners for view='{view}' (got {int(good.sum())}).")

    xs = uv[good, 0]
    ys = uv[good, 1]
    x0 = int(math.floor(xs.min())) - margin
    y0 = int(math.floor(ys.min())) - margin
    x1 = int(math.ceil(xs.max())) + margin
    y1 = int(math.ceil(ys.max())) + margin

    # clamp to image bounds
    x0 = max(0, min(x0, cam.width - 1))
    y0 = max(0, min(y0, cam.height - 1))
    x1 = max(x0 + 1, min(x1, cam.width))
    y1 = max(y0 + 1, min(y1, cam.height))
    return (x0, y0, x1, y1)  # x1,y1 exclusive
```

## A) Ground-truth decoding from BGR colors

### 1) Add a tolerance option (optional but useful)
This helps if your mask videos are compressed and colors aren’t exact. Default `0` keeps exact matching.

### 2) Replace `decode_mask()` with a color-aware version

```python
from dataclasses import dataclass

@dataclass(frozen=True)
class MaskColorSpec:
    mouse0_bgr: Tuple[int, int, int] = (25, 35, 40)
    mouse1_bgr: Tuple[int, int, int] = (40, 35, 25)
    bg_bgr: Tuple[int, int, int] = (0, 0, 0)
    tol: int = 0  # per-channel absolute tolerance

def _match_color(mask_bgr: np.ndarray, bgr: Tuple[int,int,int], tol: int) -> np.ndarray:
    b, g, r = bgr
    if tol <= 0:
        return (
            (mask_bgr[:, :, 0] == b) &
            (mask_bgr[:, :, 1] == g) &
            (mask_bgr[:, :, 2] == r)
        )
    # tolerance match
    return (
        (np.abs(mask_bgr[:, :, 0].astype(np.int16) - b) <= tol) &
        (np.abs(mask_bgr[:, :, 1].astype(np.int16) - g) <= tol) &
        (np.abs(mask_bgr[:, :, 2].astype(np.int16) - r) <= tol)
    )

def decode_mask(mask_arr: np.ndarray, color_spec: Optional[MaskColorSpec] = None) -> np.ndarray:
    """
    Returns HxW uint8 with labels:
      0 = background
      1 = mouse0
      2 = mouse1

    If mask is 3-channel: interpret as BGR color-coded mask using color_spec.
    If mask is 2D: assume already indexed labels 0/1/2 (kept as fallback).
    """
    if mask_arr.ndim == 2:
        return mask_arr.astype(np.uint8)

    if mask_arr.ndim == 3 and mask_arr.shape[2] in (3, 4):
        if color_spec is None:
            raise ValueError("Color-coded GT mask requires color_spec (mouse0_bgr/mouse1_bgr).")

        if mask_arr.shape[2] == 4:
            mask_arr = mask_arr[:, :, :3]

        # NOTE: cv2.imread returns BGR already, so mask_arr is BGR here.
        mask_bgr = mask_arr

        bg = _match_color(mask_bgr, color_spec.bg_bgr, color_spec.tol)
        m0 = _match_color(mask_bgr, color_spec.mouse0_bgr, color_spec.tol)
        m1 = _match_color(mask_bgr, color_spec.mouse1_bgr, color_spec.tol)

        out = np.zeros(mask_bgr.shape[:2], dtype=np.uint8)
        out[m0] = 1
        out[m1] = 2

        unknown = ~(bg | m0 | m1)
        if np.any(unknown):
            # Show a few offending colors for debugging
            bad_pixels = mask_bgr[unknown]
            # unique colors (cap)
            uniq = np.unique(bad_pixels.reshape(-1, 3), axis=0)
            uniq = uniq[:10]
            raise ValueError(
                "GT mask contains colors not matching bg(0,0,0), mouse0_bgr, mouse1_bgr. "
                f"First unique unknown BGR colors: {uniq.tolist()} "
                f"(tol={color_spec.tol})"
            )

        return out

    raise ValueError(f"Unsupported mask shape: {mask_arr.shape}")
```

## What “extract” is for
Yes: **extract is an optional preprocessing step** that chops videos into per-frame image files.

It exists because training directly from videos forces the DataLoader workers to do lots of:
- opening video files,
- seeking to frame `t` and `t-1`,
- decoding frames repeatedly

That can become **slow / unstable** compared to reading PNGs/JPEGs from disk, especially with multiple workers.

### What extract does in v2
For each `(rgb_video, mask_video)` pair, it writes:
- RGB frames to `out_rgb_dir/000000.png`, `000001.png`, …
- Mask frames to `out_mask_dir/000000.png`, … as **single-channel labels 0/1/2** (even if the original mask video is **color-coded**).  
This is a big win: you decode the color masks once, then training reads fast integer-label masks forever.

### Do you *need* to run extract?
Not strictly.

You should run **extract** if:
- your `manifest.csv` points to **video files** (mp4/avi) and you want faster + more deterministic training
- your mask GT is **color-coded** and might be affected by video compression; extracting converts it to clean label PNGs once

You can skip **extract** if:
- you already have RGB+mask **frame folders** (and `manifest.csv` points to those folders)
- performance is already fine and you prefer minimal preprocessing

Assistant: Here’s a practical **single-command** generator for `manifest.csv` that assumes the common layout:

- extracted RGB frame folders: `ROOT/frames/*_rgb/` (e.g. `vid001_front_rgb`)
- extracted mask frame folders: `ROOT/frames/*_mask/` (e.g. `vid001_front_mask`)

It writes: `sample_id,view,rgb_path,mask_path,camera_path,ethogram_path`

> Edit only the `ROOT=...` and (optionally) `CAM=...` path.

```bash
python -c "import csv,re; from pathlib import Path
ROOT=Path('/data'); FR=ROOT/'frames'; EH=ROOT/'ethograms'; CAM=ROOT/'cameras.json'; OUT=ROOT/'manifest.csv'
views={'top','front','side'}
def get_view(name):
    parts=re.split('[^a-zA-Z]+', name.lower())
    for v in ('top','front','side'):
        if v in parts: return v
    for v in ('top','front','side'):
        if f'_{v}_' in name.lower() or name.lower().endswith(f'_{v}_rgb') or name.lower().endswith(f'_{v}_mask'): return v

rows=[]

    base=rgb_dir.name[:-4]  # strip _rgb
    view=get_view(base)
    if view is None: 
        raise SystemExit(f'Cannot infer view (top/front/side) from folder name: {rgb_dir.name}')
    # mask folder: try base+'_mask' first, else any folder starting with base and containing 'mask'
    mask_dir=FR/(base+'_mask')
    if not mask_dir.exists():
        cands=sorted([p for p in FR.iterdir() if p.is_dir() and p.name.startswith(base) and 'mask' in p.name.lower()])
        if not cands: 
            raise SystemExit(f'No mask folder found for {rgb_dir.name} (looked for {mask_dir} or candidates starting with {base}*mask*)')

    # ethogram: pick the first CSV that starts with base (else blank)
    eg=''
```

It lets you do the CPU-only preprocessing **now** (frame extraction is often the longest I/O step) and then you can train later once torch imports. It doesn’t “solve” torch, but it prevents you from being blocked on data prep.

## Fix path I recommend: make a clean env that **does not pull conda-forge** and pin MKL if needed

Because your install output shows conda pulling lots of `conda-forge` packages, you’re likely suffering from “mixed channel” ABI issues. The cleanest fix is: **create a fresh env with override channels** so it *only* uses pytorch/nvidia/defaults.

User: What are these options for?

  --camera_crop \
  --camera_crop_margin 1

And which effect has resize_mode letterbox instead of stretch?

Assistant: ### `--camera_crop` / `--camera_crop_margin`
These are for **cropping each frame to the cage region** *before* resizing to `--image_size`, using your `cameras.json`:

- `--camera_crop`  
  Projects the **3D cage bounding box** (from `cage.origin/width/depth/height`) into the image using the camera’s **intrinsics/extrinsics** (`fx,fy,cx,cy,R,t`). Then it computes the 2D bounding rectangle of the projected cage corners and crops RGB + mask to that ROI.

  Practical effect:
  - removes empty background outside the cage (often a lot of pixels),
  - increases effective resolution on the animals,
  - can speed up training and improve segmentation by reducing irrelevant variation.

- `--camera_crop_margin 1`  
  Adds a pixel padding around that crop box (in **original image pixels**).  
  `1` is extremely tight; typical values are **10–30**. Tight margins can accidentally crop off parts of mice when they approach cage boundaries (especially if your projection is slightly off or the cage corners project close to the mice).

If you *don’t* use `--camera_crop`, the code still uses `camera_path` for the **resolution assertion**, but it won’t crop.

---

### `--resize_mode stretch` vs `letterbox`
After (optional) cropping, the image/mask must be resized to a fixed `--image_size × --image_size` for the network.

- `stretch` (default)  
  Resizes **independently in x and y** to exactly fit the target size.  
  Pros:
  - uses the whole input area (no padding),
  - simplest and often works fine.
  Cons:
  - **distorts aspect ratio** if the crop isn’t square (mice get “squashed”/“stretched”).

- `letterbox`  
  Resizes while **preserving aspect ratio**, then pads the remaining space with:
  - black pixels for RGB,
  - 0 labels (background) for the mask.
  Pros:
  - no geometric distortion (shapes preserved),
  - sometimes better if your downstream relies on shape cues (rearing silhouettes, etc.).
  Cons:
  - adds padding bands that the model sees (usually harmless but you’re giving up some effective resolution),
  - if the amount of padding differs per view/crop, it can add a mild domain cue (again usually fine).

**Rule of thumb:**
- If you enable `--camera_crop` and the resulting crop is roughly consistent/aspect-stable across videos, `letterbox` is often a good choice.
- If you want maximum pixel usage and don’t care about mild distortion, `stretch` is fine (and your labels are perfect, so the model can learn the mapping anyway).

If you want a low-risk starting point:  
**`--camera_crop` with margin ~10–20 + `--resize_mode letterbox`** tends to be robust.

User: Why is train_mIoU lower than val_mIoU? Do I have to expect that the val_loss increases while the train_loss decreases?

Assistant: Two separate things are going on, and both are normal in your setup.

## Why can `train_mIoU` be lower than `val_mIoU`?
Yes, that can happen, especially here, because the “train metric” you print is **not measured under the same conditions** as validation:

1) **Training uses augmentation; validation doesn’t.**  
Your training batches are harder (flips + brightness/contrast jitter). Validation is “clean”.

2) **Training is in `model.train()` mode; validation is in `model.eval()` mode.**  
Even though your U-Net has no dropout, `train()` vs `eval()` still matters in many models; plus the key point is: the training metric is computed while the model is actively being updated and seeing noisy/augmented inputs.

3) **Your training sampler is biased by oversampling.**  
You oversample frames where `interaction==1` or `mouse*_rear==1`. That makes training batches **harder on purpose** than validation, which is closer to the natural distribution. So it’s expected that training mIoU can be lower.

4) **Timing within the epoch.**  
The printed `train_mIoU` is averaged over batches **throughout the epoch**, including early batches before the epoch’s later updates improved the model. Validation is computed **after** the epoch finishes, using the final weights.

So `val_mIoU > train_mIoU` is not a red flag by itself.

## Do you have to expect `val_loss` to increase while `train_loss` decreases?
It’s not guaranteed, but it’s common once the model starts to **overfit**.

Typical patterns:
- **Early training:** both train_loss and val_loss go down; both mIoUs go up.
- **Later:** train_loss keeps going down, but val_loss bottoms out and can rise; val_mIoU plateaus or may slowly drop.

That’s the classic generalization gap.

### When should you worry?
- If **val_loss rises steadily for many epochs** and **val_mIoU stops improving** (or declines), you’re overfitting.
- If val metrics are noisy but the **best val_mIoU keeps improving occasionally**, it’s fine—just keep the “best checkpoint” (your script already saves `ckpt_best.pt` based on val mIoU).

### Practical advice for your case
Because your labels are perfect and the task is fairly constrained, you may see:
- very fast convergence,
- val metrics that look “too good” early,
- overfitting mostly showing up as *slight* deterioration or plateau rather than dramatic collapse.

If you want a cleaner comparison, the usual tweak is: compute a “train_eval” metric by running a pass on a held-out subset of training data **with `augment=False` and `model.eval()`**. That will typically bring train and val closer and remove the confusion.

## 1) Add one helper (once) near your resize functions

```python
def letterbox_params(in_w: int, in_h: int, out_w: int, out_h: int):
    """
    Computes letterbox resize geometry consistently.
    Returns: (nw, nh, pad_x, pad_y)
      - resized content is (nw, nh)
      - placed at (pad_x, pad_y) in the out_w/out_h canvas
    """
    scale = min(out_w / in_w, out_h / in_h)
    nw = int(round(in_w * scale))
    nh = int(round(in_h * scale))

    # Clamp in case rounding pushes by 1
    nw = max(1, min(nw, out_w))
    nh = max(1, min(nh, out_h))

    pad_x = (out_w - nw) // 2
    pad_y = (out_h - nh) // 2
    return nw, nh, pad_x, pad_y
```

```python
def resize_img(img: np.ndarray, out_hw: Tuple[int, int], mode: str) -> np.ndarray:
```
